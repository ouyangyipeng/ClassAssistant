## 环境准备

### 1. 安装 Conda 环境

```bash
conda create -n class-assistant python=3.11 -y
conda activate class-assistant
```

### 2. 安装 Python 依赖

```bash
cd api-service
pip install -r requirements.txt
```

主要依赖：
| 包名 | 用途 |
|------|------|
| fastapi + uvicorn | Web 框架 + ASGI 服务器 |
| python-pptx | PPT 文本提取 |
| pyaudio | 麦克风录音 |
| openai | LLM 调用（OpenAI Compatible） |
| dashscope | 阿里云百炼 Fun-ASR |
| websocket-client | 字节 Seed-ASR WebSocket 客户端 |
| python-dotenv | .env 环境变量加载 |

### 3. 安装前端依赖

```bash
cd app-ui
npm install
```

### 4. 安装 Rust（Tauri 需要）

```bash
winget install Rustlang.Rustup
# 安装后确认：
rustc --version
cargo --version
```

还需要 **Visual Studio Build Tools 2022**（MSVC 链接器）：
```bash
winget install Microsoft.VisualStudio.2022.BuildTools
# 安装时勾选 "C++ 桌面开发" 工作负载
```

## 配置

编辑 `api-service/.env`：

```ini
# LLM 配置（DeepSeek / OpenAI / 其他兼容 API）
LLM_BASE_URL=https://api.deepseek.com
LLM_API_KEY=sk-your-key
LLM_MODEL=deepseek-chat

# ASR 模式：mock / dashscope / seed-asr
ASR_MODE=seed-asr

# DashScope Fun-ASR（ASR_MODE=dashscope 时需要）
DASHSCOPE_API_KEY=sk-your-dashscope-key

# Seed-ASR（ASR_MODE=seed-asr 时需要）
SEED_ASR_APP_KEY=your-app-key
SEED_ASR_ACCESS_KEY=your-access-key
SEED_ASR_RESOURCE_ID=volc.seedasr.sauc.duration

# 录音参数（一般不需要改）
AUDIO_SAMPLE_RATE=16000
AUDIO_CHANNELS=1
AUDIO_CHUNK_SIZE=3200
```

### ASR 模式说明

| 模式 | 说明 | 依赖 |
|------|------|------|
| `mock` | 不录音不识别，开发 UI 时用 | 无 |
| `dashscope` | 阿里云百炼 Fun-ASR 实时语音识别 | `dashscope` 包 + DASHSCOPE_API_KEY |
| `seed-asr` | 字节跳动 Seed-ASR 大模型语音识别 | `websocket-client` 包 + SEED_ASR_* 配置 |

## 运行

### 方式一：分别启动前后端（推荐调试用）

**启动后端：**

```bash
# 方法 A：手动启动
conda activate class-assistant
cd api-service
python -m uvicorn main:app --host 127.0.0.1 --port 8765 --reload

# 方法 B：使用 bat 脚本（自动激活 conda）
api-service\start_server.bat
```

后端启动后可访问：
- 健康检查：`GET http://127.0.0.1:8765/api/health`
- API 文档：`http://127.0.0.1:8765/docs`（Swagger UI）
- 关键词查看：`GET http://127.0.0.1:8765/api/keywords`

**启动前端（仅 Vite 开发服务器，不含 Tauri 壳）：**

```bash
cd app-ui
npm run dev
# 浏览器打开 http://localhost:1420
```

### 方式二：Tauri 完整开发模式

```bash
cd app-ui
npm run tauri dev
```

这会同时启动 Vite 前端和 Tauri Rust 桌面壳。**首次编译 Rust 约需 2-3 分钟**，后续增量编译很快。

> ⚠️ Tauri 开发模式只启动前端，**后端需要单独启动**（方式一）。

## 调试

### 后端调试

**查看 API 文档：**
启动后端后访问 `http://127.0.0.1:8765/docs`，可以直接在 Swagger UI 中测试所有接口。

**用 PowerShell 测试接口：**

```powershell
# 健康检查
Invoke-RestMethod -Uri http://127.0.0.1:8765/api/health

# 启动监控
Invoke-RestMethod -Uri http://127.0.0.1:8765/api/start_monitor -Method Post

# 停止监控
Invoke-RestMethod -Uri http://127.0.0.1:8765/api/stop_monitor -Method Post

# 查看关键词
Invoke-RestMethod -Uri http://127.0.0.1:8765/api/keywords

# 更新自定义关键词
$body = @{keywords=@("张三","李四")} | ConvertTo-Json
Invoke-RestMethod -Uri http://127.0.0.1:8765/api/update_keywords -Method Post -Body $body -ContentType "application/json"

# 紧急救场
Invoke-RestMethod -Uri http://127.0.0.1:8765/api/emergency_rescue -Method Post

# 生成课后总结
Invoke-RestMethod -Uri http://127.0.0.1:8765/api/generate_summary -Method Post
```

**查看转录文件：**

```powershell
Get-Content data\class_transcript.txt
```

**ASR 调试：**

如果 ASR 不产生文本：
1. 确认麦克风权限已开启
2. 确认 `.env` 中 `ASR_MODE` 和对应 API Key 配置正确
3. 查看后端终端日志，寻找 `[DashScopeASR]` 或 `[SeedASR]` 前缀的错误信息
4. 对着麦克风说话后检查转录文件是否有新行写入

**常见错误：**

| 错误 | 原因 | 解决 |
|------|------|------|
| `Unauthorized, your api-key is invalid!` | DashScope API Key 无效 | 到百炼控制台重新获取 Key |
| `No module named 'xxx'` | 没在 conda 环境中运行 | `conda activate class-assistant` |
| `[SeedASR] connection/stream error` | Seed-ASR 连接失败 | 检查 SEED_ASR_APP_KEY / ACCESS_KEY |
| 启动监控成功但转录文件无内容 | 麦克风无输入 / ASR 返回空 | 对着麦克风说话测试 |

### 前端调试

- TypeScript 类型检查：`npx tsc --noEmit`
- Vite 构建：`npx vite build`
- 浏览器 DevTools：Tauri 开发模式下按 F12 打开

## 工作流程

```
用户点击「开始摸鱼」
    ↓
前端 → POST /api/start_monitor
    ↓
后端创建 ASR 实例（根据 ASR_MODE）→ 开启麦克风录音 → 流式推送音频到 ASR 服务
    ↓
ASR 回调返回识别文本 → 写入 data/class_transcript.txt
    ↓
检查是否命中关键词（点名/随机/抽查/...）
    ↓ 命中
通过 WebSocket 广播警报 → 前端收到 → 红色闪烁警告 + 声音
    ↓
用户点击「救场」→ POST /api/emergency_rescue
    ↓
读取最近 2 分钟转录 + PPT 资料 → 调用 LLM → 返回课堂上下文/问题/答案
```

## 关键代码修改记录

### 2026-03-05：ASR 集成

**新增文件：**
- `api-service/services/asr_service.py` — ASR 统一抽象层
  - `BaseASR`：定义 `start()` / `stop()` 接口和 `on_text(text, is_final)` 回调
  - `MockASR`：空实现
  - `DashScopeASR`：使用 `dashscope` SDK 的 `Recognition` + `RecognitionCallback`，后台线程录音推流
  - `SeedASR`：字节跳动 Seed-ASR WebSocket 二进制协议，手动构造 header、Gzip 压缩、pyaudio 录音
  - `create_asr(on_text)` 工厂函数：根据 `ASR_MODE` 返回对应实例

**修改文件：**
- `api-service/services/monitor_service.py`：
  - 移除旧的 `_monitor_loop()` 轮询和 `_mock_asr()` 占位
  - 改为使用 `create_asr()` 创建 ASR 实例
  - ASR 识别结果通过 `_on_asr_text(text, is_final)` 回调驱动
  - 使用 `asyncio.run_coroutine_threadsafe()` 从 ASR 工作线程安全地广播 WebSocket 警报
- `api-service/.env` / `.env.example`：新增 DashScope 和 Seed-ASR 配置项
- `api-service/requirements.txt`：新增 `dashscope`、`websocket-client`、`python-dotenv`
- `app-ui/src/components/ToolBar.tsx`：隐藏 file input 添加 `aria-label` 修复无障碍警告
- `api-service/start_server.bat`：新增一键启动脚本（自动激活 conda + 启动 uvicorn）
