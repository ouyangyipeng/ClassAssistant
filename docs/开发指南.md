# 上课摸鱼搭子 — 开发指南

## 项目概述

一款大学课堂辅助桌面悬浮窗应用。实时录音转文字 → 监控点名关键词 → 一键 LLM 救场 → 课后自动生成 Markdown 笔记。

### 技术栈

| 层级 | 技术 |
|------|------|
| 桌面端外壳 | Tauri 2.0 (Rust) — 无边框/透明/置顶悬浮窗 |
| 前端 UI | React 19 + TypeScript + TailwindCSS 4 + Vite 6 |
| 后端服务 | Python 3.11 + FastAPI + uvicorn |
| 语音识别 | Seed-ASR (字节跳动) / DashScope Fun-ASR (阿里云) / Mock |
| LLM 调用 | OpenAI Compatible API (支持 DeepSeek、GPT 等) |
| 音频采集 | PyAudio (PCM 16kHz mono) |
| 资料解析 | python-pptx / pypdf / python-docx |

## 配置

编辑 `api-service/.env`（参考 `.env.example`）：

```ini
# ASR 模式：mock / dashscope / seed-asr
ASR_MODE=seed-asr

# Seed-ASR（ASR_MODE=seed-asr 时需要）
SEED_ASR_APP_KEY=your-app-key
SEED_ASR_ACCESS_KEY=your-access-key
SEED_ASR_RESOURCE_ID=volc.bigasr.sauc.duration

# DashScope Fun-ASR（ASR_MODE=dashscope 时需要）
DASHSCOPE_API_KEY=sk-your-dashscope-key

# LLM 配置（DeepSeek / OpenAI / 其他兼容 API）
LLM_BASE_URL=https://api.deepseek.com
LLM_API_KEY=sk-your-key
LLM_MODEL=deepseek-chat

# 录音参数（一般不需要改）
AUDIO_SAMPLE_RATE=16000
AUDIO_CHANNELS=1
AUDIO_CHUNK_SIZE=3200
```

### ASR 模式说明

| 模式 | 说明 | 依赖 |
|------|------|------|
| `mock` | 不录音不识别，开发 UI 时用 | 无 |
| `dashscope` | 阿里云百炼 Fun-ASR 实时语音识别 | `dashscope` 包 + DASHSCOPE_API_KEY |
| `seed-asr` | 字节跳动 Seed-ASR 大模型语音识别 | `websocket-client` 包 + SEED_ASR_* 配置 |

### 支持的文件格式

| 格式 | 扩展名 | 解析库 |
|------|--------|--------|
| PowerPoint | `.pptx` | python-pptx |
| PDF | `.pdf` | pypdf |
| Word | `.docx` | python-docx |

### 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| `No module named 'xxx'` | 没在 conda 环境中运行 | `conda activate class-assistant` |
| `[SeedASR] connection/stream error` | Seed-ASR 连接失败 | 检查 SEED_ASR_APP_KEY / ACCESS_KEY |
| `Unauthorized, your api-key is invalid!` | DashScope API Key 无效 | 到百炼控制台重新获取 Key |

| Tauri 窗口无法拖动/关闭 | capabilities 权限不足 | 确认 default.json 含 allow-close/allow-start-dragging |

## 核心功能流程

### 用户工作流

1. 启动应用 → 屏幕角落出现半透明悬浮窗 (320×80)
2. 点击 **📄 上传资料** → 上传课件 (PPT/PDF/Word)
3. 点击 **🎣 开始摸鱼** → 开启语音监控
4. 老师点名时 → 🚨 红色警报弹出
5. 点击 **🆘 救场** → AI 分析课堂内容并给出参考答案
6. 下课后点击 **📝** → 自动生成课堂笔记

### 系统工作流

```
用户点击「开始摸鱼」
    ↓
前端 → POST /api/start_monitor
    ↓
后端创建 ASR 实例 → 开启麦克风录音 → 流式推送音频到 ASR 服务
    ↓
ASR 回调返回文本 → 写入 data/class_transcript.txt
    ↓
检查关键词 (点名/随机/抽查/签到/...)
    ↓ 命中
WebSocket 广播警报 → 前端 AlertOverlay → 红色闪烁 + 窗口置顶
    ↓
用户点击「救场」→ POST /api/emergency_rescue
    ↓
读取最近 2 分钟转录 + PPT 资料 → 调用 LLM → 返回上下文/问题/答案
```

### ASR 架构

```
MonitorService
    │
    ├── create_asr(on_text=_on_asr_text)
    │       ├── MockASR         (ASR_MODE=mock)
    │       ├── DashScopeASR    (ASR_MODE=dashscope)
    │       │     └── dashscope.Recognition + pyaudio 录音线程
    │       └── SeedASR         (ASR_MODE=seed-asr)
    │             └── websocket-client + 二进制协议 + pyaudio 录音线程
    │
    └── _on_asr_text(text, is_final)
          ├── 计算增量 delta（只对新文字做关键词检测）
          ├── _write_transcript()
          │     ├── 句中：原地覆盖活动行（seek + truncate）
          │     └── 句末标点：固定该行，开始新活动行
          ├── _check_keywords(delta)
          └── _broadcast_alert(alert) → WebSocket → 前端 AlertOverlay
```

#### 转录文件实时更新策略

- 每次 ASR 回调都会更新转录文件中的**活动行**（seek + truncate 原地覆盖）
- 遇到句末标点（`。？！；`）时固定该行，开始新的活动行
- 关键词检测只针对**增量文本（delta）**，避免同一关键词重复触发
- 停止监控时自动将未固定的活动行写入

#### 关键词配置

关键词从 `data/keywords.txt` 加载，每行一个，`#` 开头为注释：

```
# 监控关键词列表
点名
随机
抽查
回答
签到
```

- 启动监控时自动重载
- 运行中可调用 `POST /api/reload_keywords` 热重载
- 也可通过 `POST /api/update_keywords` 临时追加自定义关键词

## 打包发布

### 一键打包（推荐）

```powershell
.\build.ps1 v1.0.0
# 或
build.bat v1.0.0
```

脚本自动完成：
1. 同步版本号到 tauri.conf.json / package.json / Cargo.toml
2. PyInstaller 打包后端（自动清理 PATH 中的 tesseract 避免损坏 DLL）
3. Tauri 编译前端（`--no-bundle`，跳过 MSI/NSIS）
4. 组装 `release/` 目录 + 生成启动脚本
5. 压缩为 `ClassAssistant-vX.X.X-win-x64.zip`

### 单独打包前端 (Tauri → .exe)

```bash
cd app-ui
npx tauri build --no-bundle
```

产物路径：`app-ui/src-tauri/target/release/app-ui.exe`

### 单独打包后端 (PyInstaller → .exe)

```bash
conda activate class-assistant
cd api-service
pyinstaller backend.spec --clean --noconfirm
```

产物路径：`api-service/dist/class-assistant-backend/`

> 注意：`backend.spec` 中 `console=False`，后端 exe 将以无窗口模式运行。如需调试打包版，临时改为 `console=True` 即可看到日志输出。

### 发布包结构

```
ClassAssistant-vX.X.X/
├── 启动.bat                         # 一键启动脚本（后端后台运行，无多余窗口）
├── 上课摸鱼搭子.exe                  # Tauri 前端桌面窗口 (约 8.5 MB)
├── backend/                         # 后端服务目录
│   ├── class-assistant-backend.exe  # FastAPI 后端 (console=False, 无窗口)
│   ├── .env.example                 # 环境变量模板
│   └── _internal/                   # PyInstaller 运行时依赖（勿删除）
└── data/                            # 运行时数据
    └── summaries/                   # 课后笔记输出目录
```

### 用户安装使用步骤

1. **解压** zip 到任意目录（路径不要包含中文和空格最佳）
2. **配置** 首次双击 `启动.bat`，会自动创建 `backend/.env` 并打开记事本
3. **填写** API 密钥（ASR + LLM），保存后重新运行 `启动.bat`
4. **使用** 屏幕角落出现悬浮窗 → 上传课件 → 开始摸鱼

> 若仅测试 UI，将 `.env` 中 `ASR_MODE` 设为 `mock` 即可，无需真实 API 密钥。

### 常见安装问题

| 问题 | 解决 |
|------|------|
| 后端异常退出 | 检查 `backend/.env`；或在 spec 中改 `console=True` 重新打包查看日志 |
| Windows Defender 拦截 | PyInstaller/Tauri 打包产物会被误报，添加信任即可 |
| 无法录音 | Windows 设置 → 隐私 → 麦克风 → 允许桌面应用访问 |
| 前端打不开 | 确认后端已启动，访问 http://127.0.0.1:8765/docs 验证 |
| libfribidi-0.DLL 错误 | tesseract 的损坏 DLL 被加载；启动脚本已自动清理 PATH；或手动卸载/修复 tesseract |
