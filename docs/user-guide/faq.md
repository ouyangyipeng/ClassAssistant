# 常见问题

## 为什么启动后看不到反应

先检查：

- 是否真的运行了解压后的 exe，而不是压缩包内的临时程序
- Windows 是否拦截了程序首次启动
- 后端是否成功拉起

如果需要排查发布态启动问题，可查看 release/data 下的启动日志文件。

## 为什么没有告警

常见原因：

- 当前 ASR_MODE 不是你预期的模式
- 麦克风不可用
- 关键词文件为空或不包含你关心的词
- 当前转录质量太差，没有正确识别出关键词

## 为什么“救场”回答不准

常见原因：

- 最近转录内容过少或质量差
- 当前没有选中课程资料
- LLM 配置不正确
- 课堂问题本身需要更长上下文才能判断

## 为什么“老师讲到哪了”看起来太笼统

这个功能定位是快速追进度，而不是逐句转写回放。如果你需要更精细的信息，建议结合最近转录和引用资料一起看。

## 总结文件在哪里

开发态通常在项目根目录的 data/summaries，发布态通常在 release/data/summaries。

## 我能自己改关键词吗

可以。你可以：

- 直接编辑 data/keywords.txt 和 data/attention_keywords.txt
- 或通过系统提供的相关接口进行更新

## 我只想联调界面，不想接真实语音或模型

建议把 ASR_MODE 设为 mock，并使用设置面板保存。这样最适合前端交互开发和展示演示。
