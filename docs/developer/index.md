# 开发维护

这一栏集中放与联调、打包、接口、排错直接相关的内容，目标是让维护者能更快定位问题和改动入口。

## 适合谁先看

- 要本地启动前后端并联调的人
- 要检查接口、消息流和运行数据的人
- 要打正式包、处理发布问题或排查线上异常的人

## 建议阅读顺序

1. 先看开发运行，确认本地调试入口与端口。
2. 再看接口与联调，理解常用 API 与 WebSocket 链路。
3. 需要发布时再看打包发布。
4. 出现异常时，优先翻排错手册与附录资料。

## 这一栏包含什么

- 开发运行：如何启动开发模式与本地服务
- 打包发布：如何产出 release 包并验证后端健康检查
- 接口与联调：常用 API、调试入口与消息链路
- 排错手册：按症状组织的定位顺序与修复建议
