# 项目全景

这一栏先解决一个问题：课狐 ClassFox 到底在做什么，以及仓库里各部分是怎么连起来的。

## 适合谁先看

- 第一次接触项目，想先建立整体认知的读者
- 需要快速判断某个功能应该改前端、后端还是运行数据的人
- 准备继续扩展 ASR、LLM 或桌面端体验的开发者

## 建议阅读顺序

1. 从项目介绍开始，先看目标场景、功能边界与当前能力。
2. 再看系统架构，理解 Tauri、React、FastAPI、ASR、LLM 和数据文件的关系。
3. 最后看模块说明与数据配置，建立仓库目录和关键文件的定位。

## 这一栏包含什么

- 项目介绍：解释产品定位、核心价值和主要能力
- 系统架构：说明各层职责与数据流向
- 代码模块说明：快速理解前后端模块边界
- 数据与配置：集中查看运行时文件、配置项与落盘位置
